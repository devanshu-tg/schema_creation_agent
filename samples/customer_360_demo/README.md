# Customer 360 demo dataset

Generated by `build/synthesize_use_case_demos.py` (seed 42, reproducible).

## Relevant tables (5)

- `customers.csv`
- `products.csv`
- `orders.csv`
- `support_tickets.csv`
- `web_sessions.csv`

## Irrelevant tables (2)

These exist purely to test the agent's discovery-summary exclusion narrative.
Neither shares an identifier with the rest of the tables.

- `server_logs.csv` — ops log lines; no customer/product FKs
- `marketing_pixels.csv` — ad-pixel metadata; no joinable IDs

## Expected agent behavior

When uploaded to a workspace and asked about a relevant decision:

1. `list_tables` returns every CSV in the folder.
2. `summarize_discovery` returns the relevant + excluded split with reasons.
3. `match_all_patterns` ranks CUSTOMER_360 highest.
4. Agent narrates:

   > *"I see 7 datasets. 5 relevant (customers, products, orders, support_tickets, web_sessions). Excluding server_logs and marketing_pixels — they don't share IDs with the others."*
