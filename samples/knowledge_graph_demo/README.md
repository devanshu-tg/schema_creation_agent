# Knowledge Graph / GraphRAG demo dataset

Generated by `build/synthesize_use_case_demos.py` (seed 42, reproducible).

## Relevant tables (6)

- `documents.csv`
- `chunks.csv`
- `entities.csv`
- `concepts.csv`
- `mentions.csv`
- `chunk_concepts.csv`

## Irrelevant tables (2)

These exist purely to test the agent's discovery-summary exclusion narrative.
Neither shares an identifier with the rest of the tables.

- `build_logs.csv` — CI metadata; not the knowledge graph
- `font_cache.csv` — asset metadata; not the knowledge graph

## Expected agent behavior

When uploaded to a workspace and asked about a relevant decision:

1. `list_tables` returns every CSV in the folder.
2. `summarize_discovery` returns the relevant + excluded split with reasons.
3. `match_all_patterns` ranks KNOWLEDGE_GRAPH___GRAPH highest.
4. Agent narrates:

   > *"I see 8 datasets. 6 relevant (documents, chunks, entities, concepts, mentions, chunk_concepts). Excluding build_logs and font_cache — they're infrastructure artifacts."*


## Embedded signals

Chunks join through `mentions` to entities and through `chunk_concepts` to concepts — this is the classic GraphRAG retrieval shape.
