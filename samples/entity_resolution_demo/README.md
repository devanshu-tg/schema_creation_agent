Generated by `build/synthesize_use_case_demos.py` (seed 42, reproducible).

## Relevant tables (6)

- `crm_records.csv`
- `billing_records.csv`
- `support_records.csv`
- `emails.csv`
- `phones.csv`
- `addresses.csv`

## Irrelevant tables (2)

These exist purely to test the agent's discovery-summary exclusion narrative.
Neither shares an identifier with the rest of the tables.

- `country_codes_lookup.csv` — static lookup; no joinable IDs to people
- `data_dictionary.csv` — schema metadata, not data

## Expected agent behavior

When uploaded to a workspace and asked about a relevant decision:

1. `list_tables` returns every CSV in the folder.
2. `summarize_discovery` returns the relevant + excluded split with reasons.
3. `match_all_patterns` ranks ENTITY_RESOLUTION highest.
4. Agent narrates:

   > *"I see 8 datasets. 6 relevant (3 source-system records + shared identifier tables for emails / phones / addresses). Excluding country_codes_lookup and data_dictionary — they're reference tables."*
