# Cybersecurity demo dataset

Generated by `build/synthesize_use_case_demos.py` (seed 42, reproducible).

## Relevant tables (7)

- `users.csv`
- `assets.csv`
- `processes.csv`
- `ip_addresses.csv`
- `domains.csv`
- `network_flows.csv`
- `alerts.csv`

## Irrelevant tables (2)

These exist purely to test the agent's discovery-summary exclusion narrative.
Neither shares an identifier with the rest of the tables.

- `timezones.csv` — reference table, no joinable IDs
- `firewall_config.csv` — static config; not the security event graph

## Expected agent behavior

When uploaded to a workspace and asked about a relevant decision:

1. `list_tables` returns every CSV in the folder.
2. `summarize_discovery` returns the relevant + excluded split with reasons.
3. `match_all_patterns` ranks CYBERSECURITY highest.
4. Agent narrates:

   > *"I see 9 datasets. 7 relevant (users, assets, processes, ip_addresses, domains, network_flows, alerts). Excluding timezones and firewall_config — they're configuration / reference, not security events."*
