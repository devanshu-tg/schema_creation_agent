# Supply Chain demo dataset

Generated by `build/synthesize_use_case_demos.py` (seed 42, reproducible).

## Relevant tables (5)

- `suppliers.csv`
- `warehouses.csv`
- `products.csv`
- `shipments.csv`
- `orders.csv`

## Irrelevant tables (2)

These exist purely to test the agent's discovery-summary exclusion narrative.
Neither shares an identifier with the rest of the tables.

- `currency_rates.csv` — FX time series; no entity joins
- `holiday_calendar.csv` — reference dates; no IDs to graph entities

## Expected agent behavior

When uploaded to a workspace and asked about a relevant decision:

1. `list_tables` returns every CSV in the folder.
2. `summarize_discovery` returns the relevant + excluded split with reasons.
3. `match_all_patterns` ranks SUPPLY_CHAIN highest.
4. Agent narrates:

   > *"I see 7 datasets. 5 relevant (suppliers, warehouses, products, shipments, orders). Excluding currency_rates and holiday_calendar — they're reference data without joinable entity IDs."*


## Embedded signals

~8 suppliers are intentionally 'slow' (20-50 day delivery lag) — bottleneck queries should surface them.
