# Recommendation demo dataset

Generated by `build/synthesize_use_case_demos.py` (seed 42, reproducible).

## Relevant tables (4)

- `users.csv`
- `items.csv`
- `interactions.csv`
- `categories.csv`

## Irrelevant tables (2)

These exist purely to test the agent's discovery-summary exclusion narrative.
Neither shares an identifier with the rest of the tables.

- `page_load_times.csv` — perf metrics; no user/item joins
- `feature_flags.csv` — config table; not data

## Expected agent behavior

When uploaded to a workspace and asked about a relevant decision:

1. `list_tables` returns every CSV in the folder.
2. `summarize_discovery` returns the relevant + excluded split with reasons.
3. `match_all_patterns` ranks RECOMMENDATION highest.
4. Agent narrates:

   > *"I see 6 datasets. 4 relevant (users, items, interactions, categories). Excluding page_load_times and feature_flags — they're observability/config, not the data graph."*


## Embedded signals

Interactions are heavy-tailed: a handful of items get most of the engagement, which is what makes collaborative filtering interesting.
